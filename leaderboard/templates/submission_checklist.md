## 1. Required Files

### 1.1 Model Output
- [ ] **PLY File**: `output/{student_id}_hkisland.ply`
  - [ ] File is readable and not corrupted
  - [ ] Generated by OpenSplat (contains iteration comment in header)
  - [ ] File size is reasonable (< 500 MB recommended)
  - [ ] Contains all required properties (x, y, z, opacity, scale_*, rot_*, f_dc_*, f_rest_*)

- [ ] **Camera File**: `output/cameras.json`
  - [ ] Generated alongside PLY file
  - [ ] Contains valid camera intrinsics and extrinsics

### 1.2 Training Documentation
- [ ] **Training Log**: `logs/training.log`
  - [ ] Contains complete training output
  - [ ] Shows all iterations from start to finish
  - [ ] Includes loss values for analysis

### 1.3 Report
- [ ] **Report PDF**: `report/{student_id}_report.pdf`
  - [ ] Maximum 10 pages (excluding appendix)
  - [ ] Follows required structure (see below)
  - [ ] All figures are clear and readable
  - [ ] References properly formatted

### 1.4 Metadata
- [ ] **Metadata JSON**: `metadata.json`
  - [ ] All required fields completed
  - [ ] Accurate training configuration recorded
  - [ ] Results summary matches actual output
  - [ ] Valid JSON format (use validator)

## 5. Technical Verification

### 5.1 PLY File Validation
```bash
# Run this command to validate your PLY file
python scripts/evaluate_submission.py --ply_path ./output/model.ply --student_id YOUR_ID
```
- [ ] Validation passes without errors
- [ ] Gaussian count is reasonable
- [ ] File metadata is correct

### 5.2 Local Evaluation
```bash
# Run full local evaluation
python scripts/evaluate_submission.py \
    --ply_path ./output/model.ply \
    --training_log ./logs/training.log \
    --student_id YOUR_ID \
    --output ./my_evaluation.json
```
- [ ] Evaluation completes successfully
- [ ] Score is as expected
- [ ] No warnings or errors

### 5.3 File Size Check
```bash
# Check total submission size
du -sh submission/
```
- [ ] Total size < 500 MB (excluding large PLY files)
- [ ] Individual files within limits
